# dasPostgreSQL project instructions

dasPostgreSQL is the daslang binding + SQL-LINQ provider for
[PostgreSQL](https://www.postgresql.org/) (via libpq, the client C library). It
is the **networked** consumer of the provider contract defined in the daslang
repo: [modules/dasSQLITE/PROVIDER_CONTRACT.md](https://github.com/GaijinEntertainment/daScript/blob/master/modules/dasSQLITE/PROVIDER_CONTRACT.md)
— dasSQLITE (in-tree) is the reference implementation,
[dasDuckDB](https://github.com/borisbat/dasDuckDB) the embedded sibling; when
something here is underspecified, those are the closest precedent. Repo
scaffolding mirrors dasDuckDB (CI, Pages docs, CMake dual-mode).

All `.das` code uses **gen2 syntax**. Read the daslang repo's `CLAUDE.md` and
`skills/sql.md` for the `_sql` machinery; `skills/external_module_debugging.md`
for the `-load_module` dev loop.

## Modules

Registered in `.das_module`. A NEW daslib module MUST get its own line there, or
`require postgres/<name>` fails with `error[20605]`.

| Module | Require | Role |
|---|---|---|
| `postgres` | `require postgres` | raw libpq binding, generated from `pgsql/include/libpq-fe.h` (never edit `src/dasPOSTGRESQL.*` by hand except `main.cpp` / `userfn.cpp`) |
| `postgres_boost` | `require postgres/postgres_boost` | runner (`SqlRunner`), `with_postgres`/`open_postgres`, exec surface, `_sql` runtime helpers (`run_select` family, `sql_bind_to_stmt`/`sql_read_witness`), CRUD over `[sql_table]` structs, transactions |
| `postgres_provider` | `require postgres/postgres_provider` | lean registration shim for `daslib/sql_provider` — stmt-type factories, dialect hooks, capability flags. Must NOT require `postgres_boost` (cycles through `daslib/sql_boost`) |

The native module is `postgres` — `Module_dasPOSTGRESQL`,
`dasModulePostgreSQL.shared_module`, `src/dasPOSTGRESQL.*`.

## PostgreSQL vs SQLite divergences (the reason this repo exists)

- **`$n` placeholders only**: the macro layer emits portable `?` markers; the
  boost's execute path renumbers them `$1, $2, ...` in one quote-aware pass
  (`renumber_placeholders`). Don't mix dollar-quoted `$tag$...$tag$` bodies
  with `?` binds in raw SQL — the renumberer doesn't parse dollar quoting.
- **No per-index bind API**: `PgStmt` (postgres/postgres_stmt) accumulates
  text-format values + NULL flags + explicit type OIDs; `das_pq_exec_params`
  (src/dasPOSTGRESQL.userfn.cpp) marshals them into `PQexecParams` in one shot.
  Explicit OIDs (int8/float8/text/bytea) keep `$n` type inference from ever
  failing server-side.
- **Strict typing**: `SqlType.Integer` renders `BIGINT`, `Real` →
  `DOUBLE PRECISION`, `Text` → `TEXT`, `Blob` → `BYTEA`. Strict booleans too —
  the macro layer coerces bare bool column refs to `(col <> 0)` in boolean
  contexts (see `pred_to_sql_boolctx` in daslib/sql_linq).
- **No rowid / no auto-increment keyword**: `create_table` declares a single
  integer pk `GENERATED BY DEFAULT AS IDENTITY`; `insert(row) : int64` reads
  the pk back via `INSERT ... RETURNING` (`pkReport = ReturningPk`).
- **Savepoints supported**: nested `with_transaction` composes as
  SAVEPOINT/RELEASE, mirroring the SQLite reference (unlike DuckDB's
  depth-tracking).
- **No FTS5, no client UDFs** (`caps.fts5` and `caps.client_udfs` both off —
  PostgreSQL runs no client code; its full-text search is a different surface).
  `[sql_function]` in a postgres-only program is a compile-time error.
- **Text-format wire values**: doubles bind as `:.17g` (round-trip exact),
  bytea as `\x` hex both directions, native BOOLEAN columns read back as
  `'t'`/`'f'` (the int64 reader handles both).

## Dev workflow

- **Edit source at `D:\DASPKG\dasPostgreSQL`.** Run/lint/test with the module on
  the load path: `daslang -load_module D:/DASPKG/dasPostgreSQL <script>` (MCP
  tools: `load_modules: ["D:/DASPKG/dasPostgreSQL"]`). No junction or daspkg
  install needed.
- **C++ build:** `cmake -B build -DDASLANG_DIR=<daslang-root>` once, then
  `cmake --build build --config Release`. Kill `daslang`/`daslang-live` before
  relinking — they hold `dasModulePostgreSQL.shared_module` open.
- **Pure-daslib edits** (`daslib/*.das`, tests) need NO C++ rebuild — just rerun.
- **Local dev server**: the EDB zip CMake downloads on Windows also contains a
  full server. One-time: extract `pgsql/share` + `pgsql/bin` from the zip, then
  `./pgsql/bin/initdb.exe -D _pgdata -U postgres --auth=trust -E UTF8 --no-locale`.
  Start/stop: `./pgsql/bin/pg_ctl.exe -D _pgdata -o "-p 5499" -l _pgdata/server.log -w start`
  / `... stop`. Both `_pgdata/` and `pgsql/` are gitignored.

## Tests

`tests/sql_conformance/` is a copy of the daslang repo's shared provider
conformance suite with `_conformance_provider.das` rewritten for PostgreSQL
(throwaway schema per test on a shared server; `CONF_HAS_FTS5 = false`,
`CONF_HAS_CLIENT_UDFS = false`, and both file-gated tests dropped — see the
suite README's capability-skip policy). Needs a live server: `DAS_PG_CONNINFO`
env var, default `host=localhost port=5499 user=postgres dbname=postgres`. Run
from the daslang repo root:

```
daslang -load_module <this-repo> dastest/dastest.das -- --test <this-repo>/tests/sql_conformance --timeout 300
```

## Regenerating the binding (only when bumping the libpq version)

`bind/bind_postgres.das` parses `pgsql/include/libpq-fe.h` via libclang. Needs a
daslang build with `dasClangBind` enabled (see the daslang repo's
`skills/clang_bind_build.md`). From the repo root:

```
rm src/dasPOSTGRESQL.func_*.cpp   # stale higher-numbered chunks otherwise linger
daslang bind/bind_postgres.das
```

Then re-run CMake configure (the func_*.cpp glob re-evaluates) and rebuild.
**LP64 trap:** grep the regenerated `src/dasPOSTGRESQL.func_*.cpp` for
`unsigned long long` before pushing — size_t / pg_int64 / pg_usec_time_t
carriers must stay on the binder's skip list (`PQescape*`, `PQunescapeBytea`,
`lo_*`, `PQsocketPoll`, `PQgetCurrentTimeUSec`, `PQresultMemorySize`), or the
generated makeExtern fails under Linux gcc while building fine under MSVC.
Bumping libpq = update `PGSQL_VERSION` + `PGSQL_ZIP_SHA256` in CMakeLists.txt,
re-download, regen, rebuild, rerun the conformance suite.
